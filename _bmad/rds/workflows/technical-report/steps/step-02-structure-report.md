# Step 2: Structure Report

**Workflow:** technical-report
**Phase:** Report Architecture
**Duration:** 2-4 hours

---

## Objective

Design report structure with logical flow, section hierarchy, and narrative arc for technical audience.

---

## Instructions

Act as Marie. Create a detailed outline that transforms artifacts into a cohesive technical narrative.

### 1. Define Report Sections

Standard structure (adapt as needed):

1. **Executive Summary** (1 page)
   - Problem statement
   - Key findings
   - Recommendations

2. **Introduction & Objectives**
   - Business context
   - Research questions
   - Success criteria

3. **Data Sources & Preparation**
   - Data description
   - Quality assessment
   - Cleaning procedures

4. **Exploratory Data Analysis**
   - Distributions and patterns
   - Relationships and correlations
   - Key insights

5. **Methodology**
   - Feature engineering approach
   - Model selection rationale
   - Validation strategy

6. **Results**
   - Model performance comparison
   - Test set evaluation
   - Statistical tests

7. **Model Interpretation**
   - Feature importance
   - Partial dependence plots
   - SHAP values

8. **Limitations & Assumptions**
   - Data limitations
   - Model assumptions
   - Generalization boundaries

9. **Recommendations**
   - Actionable next steps
   - Model deployment readiness
   - Future improvements

10. **Appendix**
    - Full code listings
    - Additional diagnostic plots
    - Detailed methodology notes

### 2. Map Artifacts to Sections

Create `section-mapping.md`:
- Which artifacts go in which section?
- What visualizations support each point?
- Which code chunks demonstrate key steps?

### 3. Design Narrative Flow

For each section:
- Opening: What question does this section answer?
- Body: Present evidence systematically
- Closing: What should reader take away?

### 4. Create Report Outline

Generate `report-outline.md` with:
- Section titles and subsections
- Key points for each section
- Figures and tables planned
- Code chunks to include

---

## Validation Checklist

Before proceeding to Step 3:
- [ ] All 10 sections defined with purpose
- [ ] Artifacts mapped to sections
- [ ] Narrative flow logical and clear
- [ ] Outline approved by user

---

## Output

**Files created:**
- `report-outline.md` — Detailed structure
- `section-mapping.md` — Artifact assignments

**Next:** Proceed to Step 3 (Generate Report)
